# LunarMochiVox

English version is available [here](https://github.com/Secret-Society-Braid/LunarMochiVox/README.md)

LunarMochiVoxは、Discordのボイスチャットにて、テキスト読み上げを行うことを目的として作られたbot用アプリケーションです。
Text-to-Speech（TTS）エンジンを活用し、
いわゆる「無言勢」であっても円滑にボイスチャット上でのコミュニケーションを行うことできます。

また、このBotアプリケーションはDiscord側で用意されたAPI等を活用し、シンプルかつわかりやすいコマンド実行を実現しています。

## 主な機能

### テキスト読み上げ機能

マイクが使えない環境で、あなたのフレンドたちがボイスチャンネルで盛り上がっているような状況に遭遇したことはありませんか？

LunarMochiVoxは、そんなときに活躍するアプリケーションです。

手順はたった二つ。目的のボイスチャンネルに参加し、アプリケーションコマンド `vc` を実行すれば準備完了。
あとは、通常の会話のようにテキストチャンネルでメッセージを送信するだけ。
Botが自動的にあなたのメッセージを読み上げてくれます。

もちろん、Botはテキストチャンネルと紐づいているのでサーバーに加入しているメンバーであれば誰でも利用可能です。
一回のコマンド実行でその場にいるすべての人のメッセージを読み上げます。

LunarMochiVoxは、TTSエンジンとして `VOICEVOX` を使用しています。
多彩な声質と、AIによって担保された良質な音声を提供します。

## コントリビューション

プロジェクトへの貢献は大歓迎です！
このセクションでは簡単な紹介と、ビルドについてのみ記述しています。
本格的な貢献をお考えの方は、[CONTRIBUTING.md](./CONTRIBUTING_ja.md)をご参照ください。

1. このリポジトリをクローンします。
2. あなたが望む変更をローカルリポジトリにて行います。
3. `./gradlew shadowJar` を実行し、実行可能Jarファイルをビルドします。
4. ビルドされたJarファイルは、 `build/libs/LunarMochiVox-bundle.jar`に配置されているはずです。

ね、簡単でしょう？

> !Note: このアプリケーションでは、いかなる実行可能形式ファイルもビルドするよう設計していません。
> アプリケーションの実行には Java25以降のJRE環境が必要です。

### DockerとDocker Composeを使用する

DockerとDocker Composeに慣れている方には朗報です！
このアプリケーションは、`Dockerfile`を使用してビルドおよびイメージアプリケーションを作成することができます。

また、`docker-compose.yml`を使用すれば、
TTSエンジンであるVOICEVOX ENGINEとポート設定等も自動的にセットアップされます。
`docker compose up -d`を実行するだけで、アプリケーションを起動できます。

もちろん、DockerやDocker Composeを使用した場合でも、
質問等や疑問点があればお気軽にお尋ねください！
